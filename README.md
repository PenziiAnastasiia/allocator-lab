Висунуті умови та вимоги до алокатору пам’яті:

• Алокатор пам’яті використовує заголовки для блоків пам’яті для реалізації їхнього обліку. 

• Програма користувача працює в системі під управлінням ядра. 

• Ядро надає системні виклики, які дають змоги додати пам’ять в адресний простір процесу користувача, повернути частину адресного простору процесу користувача ядру, вказати ядру, що якісь сторінки пам’яті адресного простору процесу користувача не містять жодної інформації. 

• Пам’ять комп’ютера може бути сторінковою або ні (треба підтримувати обидва варіанти). 

• Алокатору пам’яті надано розмір сторінки. Якщо пам’ять комп’ютера є сторінковою, то це значення більше нуля та є ступенем двійки. 

• Алокатор пам’яті отримує пам’ять для розподілу (арени) та обліку в ядра (заборонено використовувати інший алокатор пам’яті в програмі користувача та використовувати пам’ять для розподілу та обліку в стеку процесу). 

• Алокатору пам’яті надано розмір типової арени. 

• Кількість арен та блоків у програмі потенційно не обмежена. 

• Алокатор пам’яті має бути апаратно-незалежним (не залежати від архітектури процесора). 

• Алокатор пам’яті має бути системно-незалежним (не залежати від ОС, крім функцій, які працюють з адресним простором процесу). 

• Алокатор пам’яті має працювати як з великими адресними просторами, так і з невеликими адресними просторами (кілька Кб). 

• Максимальний розмір блоку визначено системою, де виконується алокатор пам’яті. 

• Якщо якась інформація може бути закодована в іншій інформації, то використати таке кодування. 

• Функція mem_realloc() має вміти змінювати розмір блоку in-place (без переміщення вмісту блоку). 

• Покажчик, який повертає ядро в разі запиту пам’яті програмою користувача, завжди вирівняний належним чином, щоб його можна було призначити покажчику на об’єкт будь-якого типу з вимогою вирівнювання. 

• Якщо арена стає вільною, то її пам’ять треба повернути ядру. 

• Якщо якась сторінка пам’яті, яку розподіляє алокатор пам’яті, не містить жодної інформації, то алокатор пам’яті має повідомити ядро про це відповідним системним викликом. 

• Складність алгоритмів в алокаторі пам’яті має бути O(log n), треба оптимізувати алгоритми навіть якщо вони мають складність O(log n). 

• Оптимізувати розмір пам’яті, який алокатор пам’яті використовує для обліку блоків та арен. 

• Оптимізувати memory footprint. 

• Якщо треба використати якесь дерево в алокаторі пам’яті, то дозволено використати готове рішення. 

• Компілятор та програма користувача не знають про реалізацію алокатора пам’яті.
